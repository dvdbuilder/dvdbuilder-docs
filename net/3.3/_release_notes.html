DVDBuilder SDK for .NET Release Notes

-----------------------
OS / CLR Requirements
-----------------------

* DVDBuilder.clr4 - Windows 7 SP1 and above / CLR4 (.NET Framework 4.0/4.5)
* DVDBuilder.clr2 - Windows 7 SP1 and above / CLR2 (.NET Framework 2.0/3.0/3.5)

------------
Dependencies
------------
DVDBuilder.NET does not depend on the C/C++ runtime libraries.
However it uses DVDBuilder SDK for C++: either DVDBuilder.dll (x86) or DVDBuilder64.dll (x64).


-----------------
Demo Restrictions
-----------------

The following restrictions exist when operating in demo mode:

* Title length is limited to 5 minutes.

These restrictions are removed when a valid license is passed to the Library.

-------------
Version 2.2.1
-------------

Breaking Changes
----------------

* When an optical device is created (VRDevicePlugin.CreateOpticalDiscDevice) it must be explicitly specified whether an exclusive access is required. Prior to v2.2.1 an exclusive access was always assumed. It is recommended to use exclusive access when a device is used for recording. Otherwise, when a device is simply queried non-exclusive access is preferred.

* DVDBuilder.NET.2.0.dll, DVDBuilder.NET.3.0.dll and DVDBuilder.NET.3.5.dll are replaced by DVDBuilder.clr2.dll.
  You have to update your project references.

* The Video recorder plugins (VRPBDevice and VRFSDevice) are renamed: the 'U' suffix is dropped from the filename.
  You have to update the plugin names passed to VideoRecorder.LoadDevicePlugin.

New
----

* OLDDVDB-7: Online Documentation. Local help files are no longer distributed.
* OLDDVDB-8: Add functionality to turn on/off the trace log in the VideoRecorder optical device plugin (VRPBDevice).
  See VRDevicePlugin.EnableTraceLog.
* OLDDVDB-9: Allow to create a device with non-exclusive access from the VideoRecorder optical device plugin (VRPBDevice).

Fixed
-----

* OLDDVDB-6: Video recording fails on HL-DT-ST, DVD-RAM GH15L when VideoRecorder.Stop is called.
* OLDDVDB-10: Make sure that VideoRecorder writes only to aligned addresses when recording to optical device.

Version 1.14.0
--------------

New
---

* <videoObject> XML Element: Added 'aspectRatio', 'resolution' and 'frameRate' attributes. This feature should be used with caution. It was added to allow input from encoders which generate wrong MPEG-2 headers.
* <title> XML Element: The 'postNavigate' attribute is now able to reference titles from different title sets. This allows navigation across title sets.
* Ability to map MPEG audio and subtitle streams to DVD language codes.
* Ability to ready video object data from a stream provided by the application. To use this functionality your application must implement the DataStream and InputDataStreamFactory interfaces.
